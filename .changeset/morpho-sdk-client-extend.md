`MorphoClient` now exposes a `.extend()` method that lets integrators graft custom **entity classes** onto a client, mirroring the DevEx of the built-in `MorphoVaultV1` / `MorphoMarketV1` entities. Each registered class must extend the new `MorphoEntity` base (which carries the `protected readonly client` binding), and its constructor signature `(client, ...args)` becomes the call shape of `client.<name>(...args)`. `.extend()` returns a new client instance (statelessness preserved) typed as `this & ExtensionInstances<TExtension>` so successive calls accumulate. Validation is two-staged: at registration the map is checked for naming collisions (vs reserved members and previously registered extensions), identifier shape, and that values are classes extending `MorphoEntity`; at call time, methods on the entity instance that return `{ buildTx, getRequirements? }` are wrapped via a Proxy and structurally validated against the `Transaction` / `Requirement` contracts (other methods — fetchers, getters — pass through unchanged). The integrator keeps full control of `deepFreeze` and `metadata` injection. New exports: `MorphoEntity` base class, `EntityConstructor` / `ExtensionMap` / `ExtensionInstances` / `ExtensionAction` types, and the typed errors `ExtensionNameCollisionError`, `InvalidExtensionNameError`, `InvalidExtensionShapeError`, `InvalidEntityClassError`, `InvalidActionShapeError`, `InvalidTransactionShapeError`, `InvalidRequirementShapeError`.
